2. Explique a função das redes Docker. 

A função principal das redes Docker é gerenciar o isolamento e a conectividade dos contêineres.

Por padrão, contêineres são criados de forma isolada. As redes Docker existem para permitir que esses contêineres se comuniquem de maneira segura e controlada entre si, com a máquina hospedeira (host) e com a internet externa. Além da conectividade, elas garantem segurança através da segmentação: você pode colocar um banco de dados e uma aplicação web na mesma rede para que conversem, mantendo o banco de dados invisível para outras redes e para o mundo exterior.


3. Descreva, de forma conceitual, os principais tipos de redes Docker: 
○ bridge → comunicação entre contêineres no mesmo host 
○ host → contêiner compartilha a rede do host 
○ overlay → comunicação entre contêineres em múltiplos hosts 

Bridge (Ponte): É o tipo de rede padrão e o mais comum no dia a dia. Conceitualmente, ela cria uma rede virtual privada e isolada dentro do seu servidor (host). Ela serve para permitir a comunicação entre contêineres que estão rodando na mesma máquina física, fornecendo a eles IPs internos e uma resolução de nomes (DNS) automática nas redes customizadas.

Host (Hospedeiro): Neste modo, o Docker remove completamente o isolamento de rede. Conceitualmente, o contêiner compartilha a mesma interface de rede, o mesmo IP e as mesmas portas da máquina hospedeira. Se o contêiner abre a porta 80, ela é aberta diretamente na porta 80 do servidor real. É usado para aplicações que precisam de altíssima performance de rede, eliminando a camada de roteamento do Docker.

Overlay (Sobreposição): É uma rede distribuída construída "por cima" de redes físicas existentes. Conceitualmente, ela serve para conectar contêineres que estão rodando em múltiplos servidores (hosts) físicos diferentes, criando um túnel seguro para que eles conversem como se estivessem todos na mesma máquina local. É o padrão utilizado em ambientes de clusterização e alta disponibilidade (como Docker Swarm).
